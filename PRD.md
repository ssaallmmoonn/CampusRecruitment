# 基于协同过滤的校园招聘系统 - 需求分析文档

## 一、文档概述

### 1.1 文档目的

明确「基于协同过滤的校园招聘系统」的功能、非功能需求，为毕业设计开发提供清晰的开发边界和验收标准，参考智联校园招聘（https://xiaoyuan.zhaopin.com/）核心场景，适配高校毕设的开发复杂度。

### 1.2 系统定位

面向在校大学生、企业 HR、学校管理员的校园招聘平台，核心特色是基于**协同过滤算法**为学生智能推荐匹配的职位，解决校园招聘中 “学生找职位效率低、企业招不到合适人选” 的痛点。

### 1.3 核心角色

|  企业用户  |    注册登录、发布职位、管理职位、查看投递简历、筛选候选人    |
| 系统管理员 | 审核企业资质、管控职位内容、管理用户账号、查看系统数据（如职位 / 用户统计） |

特别说明一下协同过滤算法：

协同过滤算法：Python + Pandas + Scikit-learn。

这是毕设的核心，从零实现协同过滤无需复杂框架，重点是**落地简单、逻辑清晰**：

- 核心库：
    - Pandas：处理数据（如用户 - 职位行为数据的清洗、转换）；
    - Scikit-learn：提供基础的相似度计算（余弦相似度、皮尔逊相关系数），无需手写算法底层；
    - Numpy：辅助数值计算（可选，Pandas 已封装大部分功能）。
- 算法选型（适合毕设）：
    - 先实现**基于用户的协同过滤（UserCF）**：找到和目标学生行为相似的其他学生，推荐这些学生喜欢的职位；
    - 再补充**基于物品的协同过滤（ItemCF）**：找到目标学生浏览 / 投递过的职位相似的其他职位，作为推荐结果；
    - 理由：这两种算法逻辑简单，代码易实现，且能直观展示推荐效果（适合毕设答辩讲解）。
- 算法落地步骤（简化版）：
    1. 从 MySQL 读取学生 - 职位行为数据（如浏览、收藏记录）；
    2. 构建用户 - 职位评分矩阵（行为类型量化：浏览 = 1，收藏 = 3，投递 = 5，无行为 = 0）；
    3. 计算用户相似度（UserCF）或职位相似度（ItemCF）；
    4. 根据相似度筛选 Top-N 推荐职位，返回给后端接口。

## 二、功能需求

### 2.1 通用模块（所有角色）

#### 2.1.1 登录 / 注册

- 注册：

    - 学生：手机号 / 学号注册，验证短信 / 邮箱，设置密码，完善基础信息（姓名、专业、学历、毕业时间）；
    - 企业：企业名称、统一社会信用代码、联系人、手机号，上传营业执照（管理员审核后激活账号）；
    - 管理员：仅后台创建，不开放注册。

    

- 登录：

    - 支持账号密码登录，记住密码、忘记密码（短信 / 邮箱找回）；
    - 登录后根据角色跳转对应首页（学生首页 / 企业首页 / 管理员后台）。

    

- **退出登录**：清除登录状态，返回首页。

#### 2.1.2 个人中心

- 基本信息修改（头像、联系方式、密码等）；
- 角色专属信息管理（学生：简历管理；企业：企业信息完善；管理员：无额外信息）。

### 2.2 学生端核心功能（核心）

#### 2.2.1 职位浏览与搜索

- 参考智联校园招聘的职位列表页，展示职位名称、企业名称、薪资、地点、发布时间；
- 筛选功能：按行业、薪资、地点、学历要求、发布时间筛选；
- 搜索功能：支持职位名称、企业名称模糊搜索；
- 职位详情页：展示职位描述、任职要求、企业介绍，提供「收藏」「投递简历」按钮。

#### 2.2.2 简历管理

- 简历创建：填写基本信息（姓名、专业、学历）、实习 / 项目经历、技能证书、求职意向（期望职位、薪资、地点）；
- 简历编辑 / 删除：支持多份简历（如实习版、校招版），设置默认简历；
- 简历投递记录：查看投递的职位、投递时间、企业查看状态（已查看 / 未查看）。

#### 2.2.3 个性化推荐（核心亮点）

- 推荐专区：首页显著位置展示「为你推荐」职位列表（基于协同过滤算法）；
- 推荐逻辑：基于学生的「浏览 / 收藏 / 投递」行为，计算相似用户 / 相似职位，推荐 Top10 职位；
- 刷新推荐：支持手动刷新推荐列表，更新推荐结果。

#### 2.2.4 行为记录

- 查看历史浏览的职位、收藏的职位；
- 支持删除收藏 / 清空浏览记录。

### 2.3 企业端核心功能

#### 2.3.1 职位管理

- 职位发布：填写职位名称、薪资、地点、学历要求、职位描述、任职要求，提交后需管理员审核；
- 职位列表：查看已发布 / 待审核 / 已驳回的职位，支持编辑、下架、删除；
- 职位数据：查看职位的浏览量、收藏量、投递量。

#### 2.3.2 简历管理

- 投递列表：查看投递本企业职位的学生简历，支持「查看详情」「标记已读 / 有意向 / 拒绝」；
- 简历筛选：按专业、学历、求职意向等筛选简历；
- 简历导出：支持导出简历为 Excel 格式。

#### 2.3.3 企业信息管理

- 完善企业简介、行业、规模、办公地点等信息；
- 上传企业 logo、环境图片。

### 2.4 管理员端核心功能

#### 2.4.1 权限管控

- 企业审核：审核企业注册信息，通过 / 驳回（填写驳回理由）；
- 职位审核：审核企业发布的职位，过滤违规内容（如虚假薪资、违法岗位）；
- 用户管理：查看所有用户，禁用 / 启用账号，删除违规用户。

#### 2.4.2 数据统计

- 基础统计：用户总数（学生 / 企业）、职位总数、简历投递总数；
- 趋势图表：展示近 30 天职位发布量、简历投递量的趋势（用 ECharts 实现）；
- 推荐效果：统计推荐职位的点击率、投递率（可选，提升毕设亮点）。

### 2.5 算法模块（后端核心）

#### 2.5.1 数据采集

- 采集学生的行为数据（浏览、收藏、投递），量化为评分（如：浏览 = 1，收藏 = 3，投递 = 5）；
- 构建「用户 - 职位」评分矩阵（无行为则为 0）。

#### 2.5.2 协同过滤实现

- 基于用户的协同过滤（UserCF）：计算目标学生与其他学生的相似度（余弦相似度），推荐相似学生高评分的职位；
- 基于物品的协同过滤（ItemCF）：计算目标学生已交互职位与其他职位的相似度，推荐高相似度职位；
- 算法接口：封装为 FastAPI 接口（如`/api/recommend/{student_id}`），返回推荐职位列表（含职位 ID、推荐得分）。

#### 2.5.3 算法优化（可选，提升毕设质量）

- 缓存推荐结果：用 Redis 缓存推荐结果，避免重复计算（过期时间 1 小时）；
- 冷启动处理：对无行为记录的新学生，推荐热门职位（按浏览量 / 投递量排序）。

## 三、非功能需求

### 3.1 性能需求

- 页面加载时间≤3 秒；
- 推荐接口响应时间≤1 秒（算法计算简化，优先保证速度）；
- 支持至少 1000 条职位数据、500 个用户数据的正常查询。

### 3.2 兼容性需求

- 前端适配 PC 端主流浏览器（Chrome、Edge、Firefox）；
- 无需适配移动端（毕设聚焦核心功能，降低复杂度）。

### 3.3 安全性需求

- 密码加密：用户密码采用 BCrypt 加密存储，不明文保存；
- 接口鉴权：所有后端接口验证登录状态（FastAPI 用 JWT 令牌）；
- 防 SQL 注入：使用 ORM 框架（SQLAlchemy），避免原生 SQL 拼接。

### 3.4 易用性需求

- 界面参考智联校园招聘，简洁清晰，操作流程无冗余步骤；
- 关键操作（如投递简历、发布职位）给出明确的成功 / 失败提示。

## 五、页面结构参考（对标智联校园招聘）

1. 首页：轮播图（校园招聘公告）、热门职位、推荐职位、行业分类；
2. 学生登录 / 注册页：简洁表单，适配学生注册流程；

### 总结

1. 核心功能聚焦「学生端个性化推荐」，这是毕设的核心亮点，企业端 / 管理员端满足基础 CRUD 即可；
2. 算法部分优先实现**基于用户 / 物品的协同过滤**，逻辑简化（用 Scikit-learn 计算相似度），无需复杂优化；
3. 技术栈落地优先级：先打通前后端基础功能（登录、职位 CRUD），再集成算法实现推荐，最后补充统计 / 优化功能。
