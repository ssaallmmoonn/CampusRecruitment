# 基于协同过滤的校园招聘系统 - 需求分析文档

## 一、文档概述

### 1.1 文档目的

明确「基于协同过滤的校园招聘系统」的功能、非功能需求，为毕业设计开发提供清晰的开发边界和验收标准，参考智联校园招聘（https://xiaoyuan.zhaopin.com/）核心场景，适配高校毕设的开发复杂度。

### 1.2 系统定位

面向在校大学生、企业 HR、学校管理员的校园招聘平台，核心特色是基于**协同过滤算法**为学生智能推荐匹配的职位，解决校园招聘中 “学生找职位效率低、企业招不到合适人选” 的痛点。

简单说明一下协同过滤算法：（详细的开发在Recommendation那两个md文件里）

协同过滤算法：Python + Pandas + Scikit-learn。

这是毕设的核心，从零实现协同过滤无需复杂框架，重点是**落地简单、逻辑清晰**：

- 核心库：
    - Pandas：处理数据（如用户 - 职位行为数据的清洗、转换）；
    - Scikit-learn：提供基础的相似度计算（余弦相似度、皮尔逊相关系数），无需手写算法底层；
    - Numpy：辅助数值计算（可选，Pandas 已封装大部分功能）。
- 算法选型（适合毕设）：
    - 先实现**基于用户的协同过滤（UserCF）**：找到和目标学生行为相似的其他学生，推荐这些学生喜欢的职位；
    - 再补充**基于物品的协同过滤（ItemCF）**：找到目标学生浏览 / 投递过的职位相似的其他职位，作为推荐结果；
    - 理由：这两种算法逻辑简单，代码易实现，且能直观展示推荐效果（适合毕设答辩讲解）。
- 算法落地步骤（简化版）：
    1. 从 MySQL 读取学生 - 职位行为数据（如浏览、收藏记录）；
    2. 构建用户 - 职位评分矩阵（行为类型量化：浏览 = 1，收藏 = 3，投递 = 5，无行为 = 0）；
    3. 计算用户相似度（UserCF）或职位相似度（ItemCF）；
    4. 根据相似度筛选 Top-N 推荐职位，返回给后端接口。

## 二、功能需求

### 2.1 学生端核心功能（核心）

#### 2.1.0 游客模式

- 支持查看网站首页；
- 触发搜索、点击职位/企业信息等操作会提醒登录。

#### 2.1.1 登录/注册/个人中心

- 登录
    - 支持用户名+密码进行登录；
    - 登陆后跳转到学生端的页面。
- 注册
    - 填写用户名、密码、邮箱、电话号码、姓名、学校、专业、学历、毕业年份、求职意向进行注册。
- 个人中心
    - 支持修改头像、用户名、密码、姓名、学历、求职意向、学校、专业、毕业年份、电话号码、邮箱。

#### 2.1.2 招聘系统首页

- 支持搜索框搜索职位、专业、职位关键词，成功搜索后会跳转至职位搜索页；
- 开发三级分类导航：按专业筛选、按职位筛选，支持通过点击三级分类名称进行搜索；
- 首页轮播图广告展示，点击轮播图会跳转至企业自身开发的校园招聘网站；
- 首页品牌专区广告展示，点击品牌卡片会跳转至系统内该企业的详情页；
- 首页精选职位展示，系统根据学生用户的求职意向匹配9个精选职位进行展示。

#### 2.1.3 职位信息页与搜索页

- 支持搜索框模糊搜索职位名称；
- 开发职位信息卡片，展示职位名称、企业名称、薪资、工作地点、发布时间、职位性质、学历要求、经验要求等；
- 支持点击职位信息卡片和点击查看详情按钮查看职位详情，会跳转至职位详情页；
- 支持点击职位信息卡片的立即投递按钮进行简历投递；
- 开发筛选器：按专业/职位分类筛选，同时支持工作地点、职位类型、学历要求、经验要求、公司行业等筛选；
- 职位详情页：展示职位描述、薪资等信息，提供收藏和投递简历按钮，提供企业详情页入口、展示3个推荐职位。
- 企业详情页：展示企业信息、热招职位、在招职位，在招职位页开发职位分类筛选器，并展示职位详情。

#### 2.1.4 简历管理

- 展示可编辑的简历界面，支持用户添加内容；
- 简历含个人信息、个人优势、求职意向、教育经历、工作经历、项目经历、专业技能、其他技能/证书共8个部分；
- 支持多份简历管理，支持新增/删除/预览简历；
- 开发简历导航，可点击前往简历对应分区；
- 投递简历后企业端和管理员都能预览该份投递的简历。

#### 2.1.5 个性化推荐

- 首页精选职位列表根据求职意向匹配（基于协同过滤算法的ItemCF）；
- 职位详情页开发“你可能对这些职位感兴趣”；
- 职位推荐页：开发综合推荐、相似职位、相似同学都在投三种推荐功能：
    - 综合推荐：基于UserCF和ItemCF的混合权重推荐；
    - 相似职位：基于ItemCF进行推荐；
    - 相似同学都在投：基于UserCF进行推荐；
    - 一次展示20条推荐职位，支持无限滚动，往下滚动能源源不断展示推荐职位；
    - 支持筛选：工作城市、学历要求、工作性质、企业性质。


#### 2.1.6 行为记录（收藏与投递页）

- 我的收藏页：
    - 展示收藏的职位卡片，在职位卡片上开发取消收藏、立即投递、查看详情按钮；
    - 支持搜索职位名称；
    - 支持筛选出已投递/未投递的收藏职位；
    - 支持3种排序方式：投递时间、收藏时间、薪资；
    - 支持搜索工作地点进行筛选。

- 我的投递页：
    - 展示已投递的职位卡片，在职位卡片上开发投递状态、联系企业、取消投递、查看详情按钮；
    - 支持搜索职位名称；
    - 支持根据投递状态（未查看、已查看、不合适、面试中、不通过、通过共6种状态）进行筛选；
    - 支持3种排序方式：未读信息、投递时间、薪资；
    - 支持搜索工作地点进行筛选；
    - 支持与企业进行窗口聊天联系，开发未读信息提示功能。

#### 2.2.4 企业数据看板

- 1.核心看板（实时概览）

    这部分通常放在页面顶部，用大数字或仪表盘显示，让HR一眼看到目前的招聘“底盘”。

    - **累计指标：** 在招职位数、累计收到的简历总数、待处理简历数。
    - **活跃指标：** 今日新增投递、今日新增浏览。

- 2.职位效果分析（漏斗模型）

    这是数据统计的核心。你可以通过这三个指标计算出**招聘漏斗**，分析职位的吸引力。

    - **转化率分析：**
        - **兴趣率：** $收藏量 / 浏览量$（反映职位描述是否吸引人）。
        - **投递率：** $投递量 / 浏览量$（反映职位要求与学生匹配度是否合理）。
    - **热门职位排行：** 展示投递量最高的前 10 个职位，帮助企业了解哪些岗位是校园热门。
    - **冷门职位预警：** 浏览量高但投递量极低的职位（可能需要修改任职要求）。

- 3.协同过滤算法相关数据（算法反馈）

    既然你的选题特色是**协同过滤**，企业端应该能看到算法带来的收益。

    - **推荐点击率：** 统计有多少投递是通过系统“猜你喜欢”或“智能推荐”入口进来的。
    - **人才匹配度分布：** 展示投递学生中，高匹配度（算法评分 > 80分）的人数占比。这能向企业证明你的系统推荐确实有效。

- 4.学生画像（用户属性分析）

    帮助企业了解关注他们的是一群什么样的学生。

    - **专业分布：** 饼图展示投递学生的专业背景（如：计算机、电子、金融等）。
    - **学历/年级分布：** 统计本科与硕士的比例。
    - **技能关键词云：** 提取投递简历中的高频技能词（如：Java, Vue, Python），让企业知道这一届学生的技能栈偏好。

在撰写文档时，你可以强调：**“本系统通过监控浏览与投递的转化比率，为企业提供招聘决策支持，并利用协同过滤算法提升了长尾职位（冷门职位）的曝光度。”**

#### 2.3.3 数据看板

- 1.平台规模与增长（核心指标）：
    - 学生总数；
    - 企业总数；
    - 职位总数；
    - 简历投递总数。


- 2. 活跃度与粘性分析（流量监控）

    通过你设置的浏览、收藏、投递数据，计算全平台的活跃情况。

    - **DAU/MAU：** 每日/每月活跃用户数（学生和企业双端）。
    - **交互热度：** 全平台每日产生的总浏览量、总收藏量、总投递量趋势图。
  
- 3. 协同过滤算法效果监控（毕设核心亮点）

    这是证明你算法价值的关键数据。

    - **算法覆盖率：** 成功计算出推荐列表的用户占总用户数的比例。
    - **推荐转化贡献：**  
        - **计算公式：** $通过推荐位产生的投递量 / 全平台总投递量$。
        - **意义：** 如果这个比例高，说明你的协同过滤算法真正促进了双向匹配。

- 4.招聘生态平衡性（分布分析）

    防止平台出现“马太效应”（即头部企业简历爆满，小企业无人问津）。

    - **职位类别分布：** 哪些技术方向（如前端、后端、算法）的职位最多。
    - **投递热力图：** 统计不同时间段的投递频率，分析学生找工作的高峰期。
    - **供需比分析：**  
        - **最紧缺职位：** 浏览量高但投递量极低的类别。
        - **竞争最激烈职位：** 投递量/职位数 比例最高的类别。

## 三、非功能需求

### 3.1 性能需求

- 页面加载时间≤3 秒；
- 推荐接口响应时间≤1 秒（算法计算简化，优先保证速度）；
- 支持至少 1000 条职位数据、500 个用户数据的正常查询。

### 3.2 兼容性需求

- 前端适配 PC 端主流浏览器（Chrome、Edge、Firefox）；
- 无需适配移动端（毕设聚焦核心功能，降低复杂度）。

### 3.3 安全性需求

- 密码加密：用户密码采用 BCrypt 加密存储，不明文保存；
- 接口鉴权：所有后端接口验证登录状态（FastAPI 用 JWT 令牌）；
- 防 SQL 注入：使用 ORM 框架（SQLAlchemy），避免原生 SQL 拼接。

### 3.4 易用性需求

- 界面参考智联校园招聘，简洁清晰，操作流程无冗余步骤；
- 关键操作（如投递简历、发布职位）给出明确的成功 / 失败提示。

|                               | update_time |      | 最后一次更新时间                                       |

|                               | similarity  |      | 两个用户之间的相似度                     |
|                               | co_cnt      |      | 两个用户共同对其产生过交互行为的职位数量 |
|                               | update_time |      | 最后一次更新时间                         |

**用户-岗位交互行为的聚合评分表**：它将用户对某个岗位的多种行为（浏览、收藏、投递）汇总成一个统一的分值，作为推荐算法的核心输入。

| 表名                         | 字段名           | 类型 | 说明                                          |
| ---------------------------- | ---------------- | ---- | --------------------------------------------- |
| recommendation_useritemscore | id               |      | 主键，自增                                    |
|                              | score            |      | 综合计算后的分数，权重为：浏览1，收藏3，投递5 |
|                              | view_cnt         |      | 浏览次数                                      |
|                              | collect_cnt      |      | 收藏次数                                      |
|                              | deliver_cnt      |      | 投递次数                                      |
|                              | last_action_time |      | 最后交互时间                                  |
|                              | job_id           |      | 职位id                                        |
|                              | student_id       |      | 学生id                                        |



**综合推荐表**：对候选群进行加权混合后得出的推荐职位表。

| 表名                          | 字段名       | 类型 | 说明                                      |
| ----------------------------- | ------------ | ---- | ----------------------------------------- |
| recommendation_recommendation | id           | INT  | 主键，自增                                |
|                               | job_id       |      | 推荐的职位的id                            |
|                               | student_id   |      | 推荐给某位学生的id                        |
|                               | reason       | JSON | 推荐理由                                  |
|                               | score_hybrid |      | 最终的加权混合分数，0.6*ItemCF+0.4*UserCF |
|                               | score_item   |      | Item-based分数                            |
|                               | score_user   |      | User-based分数                            |
|                               | update_time  |      | 最后一次更新时间                          |
